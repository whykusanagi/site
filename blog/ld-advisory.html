Why This Is Structurally Different

LD Cloud's underlying product is a cloud Android emulator. The Android instance does not run on the user's PC — it runs on the operator's servers, and the user receives a rendered video stream of it.

The consequence most people don't see: when a user types a password into a game on this product, the keystrokes are not on the user's machine. They are on the operator's server. The operator's server takes the keystrokes and draws the asterisks the user sees. The user's screen only ever shows asterisks — but the operator's machine held the plaintext keystrokes for the brief moment between keypress and render.

// The Line

Recordings showing only asterisks are not exculpatory. The capture moment exists structurally, by design of the product. Whether keystrokes were saved alongside the videos is a retention question. That they were on the operator's machine is not in question.

The Sogou IME Angle (Local LDPlayer)

Most users who say "LDPlayer" mean the PC emulator running on their own machine, not the cloud product. The cloud-architecture argument doesn't apply to that group directly — but there's a separate concern.

The same install bundle ships LD's own client app and Sogou Pinyin, Tencent's Chinese input method. Co-bundling is what you'd do if Sogou were being preinstalled as the default keyboard inside the LDPlayer Android image. If so, every keystroke a local LDPlayer user types hits the Sogou input method first. The IME sees plaintext. The game's password field only sees the result. Sogou has a long, well-documented telemetry history including cloud completion that uploads typed input to remote servers.

Honest caveat: I haven't visually confirmed Sogou is the default IME in the local LDPlayer image. The co-bundling makes it the obvious read, but if asked in chat, the truthful answer is "most likely, but not personally verified."

Q&A — Safe Answers

"Is this LDPlayer recording me right now?"

If you're on the cloud streaming product, sessions run on the operator's servers, and the buckets show those servers produce recordings. Whether your specific session is in there is something only the operator knows. Defensive assumption: yes. Treat your credentials as exposed.

"Is Tencent involved?"

Tencent owns Sogou. Sogou Pinyin appears to ship in the LDPlayer bundle. I have no evidence Tencent operates LD Cloud itself. Don't conflate the IME question with the operator question.

"Is this a state spying operation?"

No evidence either way. Operator is mainland-China based, buckets are on Huawei Cloud which is also mainland-China — circumstantial. Not speculating on stream.
